PALIHOG App Documentation

PALIHOG App is a campus-based, real-time task-management mobile application designed for students across various colleges and universities. Originally developed as a capstone project, the app connects students who need help with tasks (like purchasing items, delivering documents, or running errands) to other students who can fulfill those requests for a fee. The platform features an intuitive mobile app for users and a robust web portal for administrative management.

Technical Architecture

Mobile App Stack

  • Core: React Native (v0.79) & Expo (v53).
  • Navigation: Expo Router & React Navigation for deep-linking.
  • UI/Animations: react-native-reanimated for native-driven smoothness.
  • Integrations: Geolocation via Maps, native media handling using Expo ecosystem.

Web Admin Stack

  • Framework: Built as a Modern SPA using React 19 and Vite 7.
  • State & Routing: Inertia.js v2 bridging React seamlessly to the backend.
  • UI/UX: Tailwind CSS v4, Material UI (MUI) v7, and Radix UI primitives.
  • Architecture: Hybrid SSR-ready (build:ssr) for optimized FCP.

Unified Backend Infrastructure

  • Primary Engine: Laravel 12 (PHP 8.2+) powering robust business logic, queues, and headless operations.
  • Database: Firebase v12 (Firestore) for real-time document storage, bidding logic, and messaging threads.
  • Real-time Systems: Custom targeted notification broadcasting and secure live location-sharing endpoints.

Security Implementations

  • RBAC Rules: Strict Firestore admin separation (rules_version = '2') to completely protect data and real-time bidding constraints.
  • Web Security: Built-in CSRF protection via Laravel/Inertia, strict XSS sanitization via React, and protected middleware routes.
  • Headless Authentication: Laravel Fortify handles secure backend authentication completely separate from the UI.
  • Data Integrity: Soft deletion patterns for messaging to preserve context, and mathematical field-level validation for monetary systems.
  • Secure Auth Flows: Rate-limited database rules for email OTPs and password recovery paths to prevent brute-force attacks.
